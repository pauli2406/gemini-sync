IngestRelay - Contributor and Agent Contract

Purpose
- Define the default operating rules for contributors and AI agents.
- Keep this repository focused on runtime/tooling plus curated connector examples.

Operating Model
1. Runtime/tooling repository
   - Contains runtime code, docs, tests, scripts, CI, and curated examples under connectors/.
2. Connector-config repository (recommended)
   - Contains environment/team-specific connector YAML files.
3. Discovery behavior
   - API/Ops/Studio discover connectors from CONNECTORS_DIR.
   - If CONNECTORS_DIR is unset, discovery defaults to connectors/ in this repository.

Non-Negotiable Rules
- Do not commit customer-specific or environment-specific connectors to this runtime repo.
- Preserve schema and API compatibility unless an explicit migration plan exists.
- Keep generated references committed and in sync:
  - docs/reference/connector-fields.md
  - website/static/openapi.json

PR Evidence Expectations
- Red evidence:
  - failing or mismatch proof for new behavior/gates before implementation
- Green evidence:
  - passing tests/evals after implementation
- Docs evidence:
  - updated docs mapped by docs/doc_sync_map.yaml
- Risk evidence:
  - risk tier assignment with required gate outcomes
